UseCase-схема: Управление персоналом и задачами
Акторы (Roles):
Администратор:

Создаёт учетные записи сотрудников.

Назначает роли сотрудникам.

Назначает задачи.

Просматривает список задач и отчетов.

Отмечает задачи как выполненные (опционально, если задачи требуют подтверждения).

Сотрудник:

Выполняет задачи, назначенные администратором.

Отмечает задачи как выполненные.

Основные UseCases:
Управление сотрудниками (Администратор)

Создание учетной записи сотрудника.

Ввод имени, роли, начальных задач.

Сохранение данных в таблице staff.

Изменение учетной записи сотрудника.

Редактирование имени, роли, списка задач.

Обновление данных в таблице staff.

Удаление учетной записи сотрудника.

Удаление строки из таблицы staff на основании staff_id.

Назначение задач (Администратор)

Создание задачи.

Выбор сотрудника из списка (staff_id).

Указание описания задачи и её статуса (по умолчанию pending).

Сохранение данных в таблице task_reports.

Просмотр списка задач.

Отображение задач с объединением данных из таблиц task_reports и staff, чтобы показать имя сотрудника.

Завершение задачи.

Изменение статуса задачи на completed и запись времени выполнения.

Выполнение задач (Сотрудник)

Получение списка задач.

Просмотр задач, которые назначены конкретному сотруднику, с указанием их статуса.

Завершение задачи.

Изменение статуса задачи на completed.

Отчетность (Администратор)

Просмотр отчета по задачам:

Общее количество задач, выполненных и невыполненных.

Анализ задач по сотрудникам.

Просмотр отчетов по бронированиям:

Активные бронирования.

Статистика популярных номеров.

Загруженность номеров (доступные/недоступные).

Дополнительные детали:
Связи между UseCases:
Администратор может добавлять задачи и назначать их только сотрудникам, зарегистрированным в системе (staff).

Задачи связаны с сотрудником через staff_id.

Отчеты зависят от данных из других таблиц (bookings, rooms, task_reports).

Таблицы для реализации:
staff: хранит информацию о сотрудниках.

task_reports: хранит назначенные задачи и их статус.

bookings, rooms: используются для отчетов по бронированиям.
